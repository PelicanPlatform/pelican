## Commands Reference

Pelican is distributed as two separate executables:

| Binary | Purpose |
|---|---|
| `pelican` | Client-side tools — transfer data, manage credentials, submit jobs |
| `pelican-server` | Server-side tools — run origins, caches, directors, and registries |

Both binaries share a common set of commands (`config`, `credentials`, `namespace`, `token`).

---

### pelican — client binary

The `pelican` binary is for users who need to **access data** from a federation or manage client-side credentials.

[Browse the pelican command reference →](/commands-reference/pelican/)

Key subcommands:

* [`pelican object`](/commands-reference/pelican/object/) — get, put, copy, and delete objects
* [`pelican job`](/commands-reference/pelican/job/) — manage asynchronous transfer jobs
* [`pelican plugin`](/commands-reference/pelican/plugin/) — HTCSS plugin integration
* [`pelican rclone`](/commands-reference/pelican/rclone/) — rclone integration

---

### pelican-server — server binary

The `pelican-server` binary is for operators who need to **run federation services**.

[Browse the pelican-server command reference →](/commands-reference/pelican-server/)

Key subcommands:

* [`pelican-server origin`](/commands-reference/pelican-server/origin/) — run an origin service
* [`pelican-server cache`](/commands-reference/pelican-server/cache/) — run a cache service
* [`pelican-server director`](/commands-reference/pelican-server/director/) — run a director service
* [`pelican-server registry`](/commands-reference/pelican-server/registry/) — run a registry service
* [`pelican-server key`](/commands-reference/pelican-server/key/) — manage issuer keys
* [`pelican-server generate`](/commands-reference/pelican-server/generate/) — generate server credentials
